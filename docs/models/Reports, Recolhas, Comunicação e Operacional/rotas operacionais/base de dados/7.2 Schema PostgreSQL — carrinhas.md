# Schema PostgreSQL — `carrinhas` (RF-28)

Frota de veículos de recolha. Gerida pelo **Gestor/Admin** ([[02-Requisitos/M11-Frota-Equipas|RF-28]]). Uma carrinha é atribuída a uma [[7.3 Schema PostgreSQL — equipas_rota|equipa de rota]] para executar uma recolha.

```
┌─────────────────────────────────────────────────────────────────────────────┐
│  tabela: carrinhas                                                          │
├──────────────────────────┬──────────────────────┬───────────────────────────┤
│ coluna                   │ tipo                 │ constraints / notas       │
├──────────────────────────┼──────────────────────┼───────────────────────────┤
│ id                       │ UUID                 │ PK, gen_random_uuid()     │
│ matricula                │ VARCHAR(16)          │ NOT NULL, UNIQUE          │
│ tipo                     │ VARCHAR(20)          │ NOT NULL                  │
│                          │                      │ CHECK IN (                │
│                          │                      │  'LIGEIRA','PESADA',      │
│                          │                      │  'SATELITE')              │
│ capacidade_kg            │ INTEGER              │ NOT NULL, CHECK > 0       │
│ zona_base_id             │ UUID                 │ nullable                  │
│                          │                      │ FK → zonas(id)            │
│                          │                      │ base operacional          │
│ estado                   │ VARCHAR(20)          │ NOT NULL                  │
│                          │                      │ DEFAULT 'DISPONIVEL'      │
│                          │                      │ CHECK IN (                │
│                          │                      │  'DISPONIVEL',            │
│                          │                      │  'EM_ROTA',               │
│                          │                      │  'MANUTENCAO')            │
│ notas                    │ TEXT                 │ nullable                  │
│ criado_em                │ TIMESTAMPTZ          │ NOT NULL, DEFAULT now()   │
│ atualizado_em            │ TIMESTAMPTZ          │ NOT NULL, DEFAULT now()   │
│ eliminado_em             │ TIMESTAMPTZ          │ nullable (soft delete)    │
└──────────────────────────┴──────────────────────┴───────────────────────────┘
```

**Índices:**

```
┌──────────────────────────────────────┬────────────────┬──────────────────────┐
│ Coluna(s)                            │ Tipo           │ Uso                  │
├──────────────────────────────────────┼────────────────┼──────────────────────┤
│ (matricula)                          │ UNIQUE BTREE   │ unicidade do veículo │
│ (zona_base_id, estado)               │ BTREE          │ frota disponível por │
│                                      │                │ zona para montar     │
│                                      │                │ equipa               │
│ (estado) PARTIAL WHERE               │ BTREE          │ carrinhas livres     │
│   estado = 'DISPONIVEL'              │                │                      │
└──────────────────────────────────────┴────────────────┴──────────────────────┘
```

**Notas de design:**
- Apenas carrinhas com `estado = 'DISPONIVEL'` podem ser atribuídas a uma nova `equipas_rota` (validado no serviço, RF-29).
- Ao ativar uma equipa, a carrinha passa a `EM_ROTA`; ao concluir/cancelar a rota, volta a `DISPONIVEL`.
- Criação/edição exclusiva de `GESTOR`/`ADMIN`; toda a alteração de estado é auditada (`audit_log`, RNF-SEG-03).

## Relacionamentos

```
carrinhas (N:1) zonas              FK: carrinhas.zona_base_id
carrinhas (1:N) equipas_rota       FK: equipas_rota.carrinha_id
```
