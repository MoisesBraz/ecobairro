# Schema PostgreSQL — `equipas_rota` (RF-29)

Equipa de recolha montada pelo **Gestor**: associa um ou mais **operadores** (terreno) a uma **carrinha**, numa **zona**, para executar uma **rota** ([[02-Requisitos/M11-Frota-Equipas|RF-29]]). É o elo entre o planeamento (Gestor) e a execução física (Operador, [[7.1 Schema PostgreSQL — rotas_execucao|rotas_execucao]]).

```
┌─────────────────────────────────────────────────────────────────────────────┐
│  tabela: equipas_rota                                                       │
├──────────────────────────┬──────────────────────┬───────────────────────────┤
│ coluna                   │ tipo                 │ constraints / notas       │
├──────────────────────────┼──────────────────────┼───────────────────────────┤
│ id                       │ UUID                 │ PK, gen_random_uuid()     │
│ gestor_id                │ UUID                 │ NOT NULL                  │
│                          │                      │ FK → users(id)            │
│                          │                      │ quem criou a equipa RF-29 │
│ carrinha_id              │ UUID                 │ NOT NULL                  │
│                          │                      │ FK → carrinhas(id)        │
│ operadores               │ UUID[]               │ NOT NULL, CHECK len >= 1  │
│                          │                      │ FK lógico → users(id)     │
│                          │                      │ (role = OPERADOR)         │
│ zona_id                  │ UUID                 │ NOT NULL                  │
│                          │                      │ FK → zonas(id)            │
│ rota_id                  │ UUID                 │ nullable                  │
│                          │                      │ FK → rotas_execucao(id)   │
│                          │                      │ preenchido ao iniciar     │
│ estado                   │ VARCHAR(20)          │ NOT NULL                  │
│                          │                      │ DEFAULT 'PLANEADA'        │
│                          │                      │ CHECK IN (                │
│                          │                      │  'PLANEADA','ATIVA',      │
│                          │                      │  'CONCLUIDA','CANCELADA') │
│ turno                    │ VARCHAR(20)          │ nullable (MANHA/TARDE)     │
│ criado_em                │ TIMESTAMPTZ          │ NOT NULL, DEFAULT now()   │
│ atualizado_em            │ TIMESTAMPTZ          │ NOT NULL, DEFAULT now()   │
└──────────────────────────┴──────────────────────┴───────────────────────────┘
```

**Índices:**

```
┌──────────────────────────────────────┬────────────────┬──────────────────────┐
│ Coluna(s)                            │ Tipo           │ Uso                  │
├──────────────────────────────────────┼────────────────┼──────────────────────┤
│ (operadores)                         │ GIN            │ "equipas deste       │
│                                      │                │ operador" via @>     │
│ (zona_id, estado)                    │ BTREE          │ equipas ativas/zona  │
│ (carrinha_id)                        │ BTREE          │ ocupação da carrinha │
│ (gestor_id, criado_em DESC)          │ BTREE          │ histórico do gestor  │
└──────────────────────────────────────┴────────────────┴──────────────────────┘
```

**Notas de design:**
- `operadores UUID[]` permite equipas de 1+ trabalhadores; query típica: `WHERE operadores @> ARRAY[:user_id]::uuid[]` (índice GIN) para o operador ver as suas equipas.
- Ao criar a equipa (RF-29), o serviço valida que a **carrinha está `DISPONIVEL`** e que **todos os operadores têm `operador_perfis.disponivel = true`**; depois marca a carrinha `EM_ROTA` e notifica os operadores (BullMQ).
- Ao concluir/cancelar, a carrinha volta a `DISPONIVEL` e os operadores ficam livres.
- `rota_id` liga 1:1 à `rotas_execucao` gerada quando o operador aceita.

## Relacionamentos

```
equipas_rota (N:1) users (gestor_id)
equipas_rota (N:1) carrinhas        FK: equipas_rota.carrinha_id
equipas_rota (N:1) zonas            FK: equipas_rota.zona_id
equipas_rota (N:M) users            via operadores[] (role = OPERADOR)
equipas_rota (1:1) rotas_execucao   FK: equipas_rota.rota_id
```
