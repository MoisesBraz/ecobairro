# Schema PostgreSQL — `operador_perfis` (terreno, RF-24)

Perfil do **Operador de terreno** (motorista/recolha). Distinto do [[2.10 Schema PostgreSQL — gestor_perfis|`gestor_perfis`]] (backoffice): o operador **executa** rotas, não as planeia. Relação 1:1 com `users WHERE role = 'OPERADOR'`.

```
┌─────────────────────────────────────────────────────────────────────────────┐
│  tabela: operador_perfis                                                    │
│  Relação: 1:1 com users WHERE role = 'OPERADOR'                             │
│  Contém: carta de condução, zona-base, disponibilidade                      │
├──────────────────────────┬──────────────────────┬───────────────────────────┤
│ coluna                   │ tipo                 │ constraints / notas       │
├──────────────────────────┼──────────────────────┼───────────────────────────┤
│ id                       │ UUID                 │ PK, gen_random_uuid()     │
│ user_id                  │ UUID                 │ NOT NULL, UNIQUE          │
│                          │                      │ FK → users(id)            │
│                          │                      │ ON DELETE CASCADE         │
│ nome_completo            │ VARCHAR              │ NOT NULL                  │
│ entidade_empregadora     │ VARCHAR(100)         │ NOT NULL                  │
│                          │                      │ ex: 'VEOLIA'              │
│ carta_conducao           │ VARCHAR(20)          │ nullable                  │
│                          │                      │ categoria/nº (terreno)    │
│ zona_base_id             │ UUID                 │ nullable                  │
│                          │                      │ FK → zonas(id)            │
│ disponivel               │ BOOLEAN              │ NOT NULL, DEFAULT true     │
│                          │                      │ elegível p/ equipa RF-29  │
│ criado_em                │ TIMESTAMPTZ          │ NOT NULL, DEFAULT now()   │
│ atualizado_em            │ TIMESTAMPTZ          │ NOT NULL, DEFAULT now()   │
└──────────────────────────┴──────────────────────┴───────────────────────────┘
```

**Índices:**

```
┌──────────────────────────────────────┬────────────────┬──────────────────────┐
│ Coluna(s)                            │ Tipo           │ Uso                  │
├──────────────────────────────────────┼────────────────┼──────────────────────┤
│ (user_id)                            │ UNIQUE BTREE   │ join rápido com users│
│ (zona_base_id, disponivel)           │ BTREE          │ operadores livres por│
│                                      │                │ zona p/ montar equipa│
│ (nome_completo)                      │ GIN pg_trgm    │ pesquisa por nome    │
└──────────────────────────────────────┴────────────────┴──────────────────────┘
```

**Notas de design:**
- `disponivel` é consultado pelo Gestor ao criar uma equipa (RF-29): só operadores `disponivel = true` na zona podem ser atribuídos.
- Operadores são criados exclusivamente por `ADMIN` via `POST /admin/operadores` — não há auto-registo.
- O soft delete do operador é controlado por `users.eliminado_em`.

## Relacionamentos

```
operador_perfis (1:1) users        FK: operador_perfis.user_id
operador_perfis (N:1) zonas        FK: operador_perfis.zona_base_id
users (operador) (N:M) equipas_rota via equipas_rota.operadores[]
```
