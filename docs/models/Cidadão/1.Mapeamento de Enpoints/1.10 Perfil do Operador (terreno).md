# Perfil do Operador (terreno, self-service)

Endpoints para o **Operador de terreno** (`OPERADOR`) gerir o seu próprio perfil e disponibilidade. Distinto do [[1.9 Perfil do Gestor|perfil do Gestor]] (backoffice). Tabela: [[2.11 Schema PostgreSQL — operador_perfis]].

| # | Método | Rota | Descrição | Auth | Fluxo |
|---|--------|------|-----------|------|-------|
| POp1 | `GET` | `/operadores/me` | Obter perfil próprio do operador | OPERADOR, ADMIN | NestJS → Redis (`operador:profile:{id}`) ou PG (JOIN users + operador_perfis) |
| POp2 | `PUT` | `/operadores/me` | Atualizar campos editáveis (phone) | OPERADOR, ADMIN | NestJS → PG write → NOTIFY operador_updated → Redis invalidar |
| POp3 | `PATCH` | `/operadores/me/disponibilidade` | Marcar-se disponível/indisponível | OPERADOR | NestJS → PG write `operador_perfis.disponivel` → NOTIFY |

**Resposta de POp1:**

```
{
  id: uuid (users.id),
  email: string,
  phone: string | null,
  role: "OPERADOR",
  nome_completo: string,
  entidade_empregadora: string,
  carta_conducao: string | null,
  zona_base: { id: uuid, nome: string } | null,
  disponivel: boolean,
  criado_em: ISO8601
}
```

**Corpo de POp3 (disponibilidade):**

```
{ disponivel: boolean }
```

Nota: `nome_completo`, `entidade_empregadora`, `carta_conducao`, `zona_base_id` e `role` só podem ser alterados pelo `ADMIN` via `PUT /admin/operadores/:id`. O operador edita apenas `phone` e a sua `disponivel`.

**Cache Redis:**

```
operador:profile:{user_id}  TTL: 30min
  Invalidação: NOTIFY operador_updated → DEL operador:profile:{id}
```
