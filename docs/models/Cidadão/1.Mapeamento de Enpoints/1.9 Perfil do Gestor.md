#### Perfil do Gestor (self-service)

Endpoints para gestores (GESTOR, ADMIN) gerirem o seu próprio perfil.

| #   | Método | Rota              | Descrição                                    | Auth                                                              | Fluxo                                                              |
| --- | ------ | ----------------- | -------------------------------------------- | ----------------------------------------------------------------- | ------------------------------------------------------------------ |
| PO1 | `GET`  | `/gestores/me`  | Obter perfil próprio do gestor             | GESTOR, ADMIN          | NestJS → Redis hit (gestor:profile:{id}) ou PG réplica (JOIN users + gestor_perfis) |
| PO2 | `PUT`  | `/gestores/me`  | Atualizar campos editáveis (cargo, phone)    | GESTOR, ADMIN          | NestJS → PG write users (phone) + gestor_perfis (cargo) → NOTIFY gestor_updated → Redis invalidar |

**Gestão do quiz (implementado).** O gestor (e admin) gerem o banco de
perguntas do quiz via `GET/POST/PATCH/DELETE /api/v1/admin/quiz/perguntas`
(`assertManager`; 1 opção correta, 2-6 opções; auditado). UI em
`/gestao-quiz`. Detalhe em
[[models/Ecopontos, Zonas, Badges e Quiz/quiz/Gestão (admin)]] e
`apps/api/src/gamification/CLAUDE.md`.

**Resposta de PO1:**

```
{
  id: uuid (users.id),
  email: string,
  phone: string | null,
  role: string,
  email_verified: boolean,
  nome_completo: string,
  entidade_empregadora: string,
  cargo: string | null,
  zonas_responsabilidade: [
    { id: uuid, nome: string }
  ],
  criado_em: ISO8601
}
```

**Corpo de PO2 (Atualizar perfil gestor):**

```
{
  phone: string (optional),
  cargo: string (optional)
}
```

Nota: Os campos `nome_completo`, `entidade_empregadora`, `zonas_responsabilidade` e `role` só podem ser alterados pelo ADMIN via `PUT /admin/gestores/:id` (PA8). O gestor pode apenas editar `phone` (em `users`) e `cargo` (em `gestor_perfis`).

**Cache Redis:**

```
gestor:profile:{user_id}  TTL: 30min
  Campos: email, phone, role, nome_completo, entidade_empregadora,
          cargo, zonas_responsabilidade (com nomes resolvidos)
  Invalidação: NOTIFY gestor_updated → DEL gestor:profile:{id}
```
